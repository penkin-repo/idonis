## 🏛️ Ключевые архитектурные решения

| Дата       | Решение | Причина |
|------------|---------|---------|
| 2026-03-15 | Firebase Functions → **Render.com** | Firebase Functions требует Blaze (карту), Render бесплатно без карты |
| 2026-03-15 | Firebase Scheduler → **APScheduler** | Внутри Render-сервиса, не нужны внешние cron сервисы |
| 2026-03-15 | Firebase Secrets → **Render Env Vars** | Дашборд Render, так же безопасно |
| 2026-03-15 | + **UptimeRobot** | Keep-alive, бесплатно, пингует /health каждые 5 мин |
| 2026-03-15 | Webhook вместо polling | Меньше задержка, нет постоянного запроса, Firebase Scheduler для cron |
| 2026-03-15 | OpenRouter вместо прямого OpenAI | Гибкость смены модели через env, доступ к Claude/Gemini/GPT |
| 2026-03-15 | Python 3.11 для Functions | Нативная поддержка Firebase, хорошая экосистема для AI |
| 2026-03-15 | Firestore (NoSQL) | Гибкая схема, Firebase-нативный выбор, realtime |
| 2026-03-15 | pnpm | Глобальный стандарт пакетных менеджеров в проектах команды |
